# COVID-19 Weekly Analytics Demo

A Dagster ETL pipeline that generates synthetic COVID-19 data and a FastAPI endpoint that serves aggregated analytics.

## Project Structure

```
dag/
  pipeline.py       # Dagster ETL pipeline (extract, transform, load)
  main.py           # FastAPI app serving analytics from the pipeline output
  covid.db          # SQLite database (auto-generated by the pipeline)
  requirements.txt  # Python dependencies
```

## Setup

```bash
pip install -r dag/requirements.txt
```

## Running the Pipeline

### One-off execution

```bash
cd dag
python3 -c "from pipeline import covid_etl_job; covid_etl_job.execute_in_process()"
```

This runs the full ETL:

1. **Extract** - Generates synthetic daily case data for Egypt, Ethiopia, Kenya, and Nigeria (120 days).
2. **Transform** - Melts the wide-format data and resamples into weekly totals.
3. **Load** - Writes the result into the `covid_weekly` table in `covid.db`.

### With the Dagster UI

```bash
cd dag
dagster dev -f pipeline.py
```

Open http://localhost:3000 to view and trigger the pipeline. A daily schedule (1 AM) is pre-configured.

## Running the API

```bash
cd dag
uvicorn main:app --reload
```

### Example request

```
GET /analytics/covid?country=Ethiopia&start=2026-03-01&end=2026-06-25
```

### Example response

```json
[
  {
    "country": "Ethiopia",
    "total_cases": 47034.0
  }
]
```

API docs are available at http://localhost:8000/docs.

## How It Works

```
extract_covid()  -->  transform_covid()  -->  load_covid_weekly()
  (synthetic          (melt + weekly            (write to
   daily data)          resample)                SQLite)
       |                                           |
       |                                           v
       |                                      covid.db
       |                                           |
       |                                           v
       |                                    FastAPI  /analytics/covid
```

The pipeline generates fresh data on each run and replaces the `covid_weekly` table. The FastAPI app reads from that table to serve queries filtered by country and date range.
